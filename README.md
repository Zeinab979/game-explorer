🎮 GameExplorerGameExplorer is a modern Android application built to explore the vast world of video games using the RAWG Video Games Database API. The project is built following Clean Architecture principles to ensure scalability, testability, and maintainability.🚀 Features•Game Discovery: Browse a paginated list of trending games.•Instant Search: Filter games locally from the loaded list.•Deep Details: View comprehensive information about specific games, including descriptions and ratings.•Resilient Networking: Handles API errors (like Internal Server Errors) gracefully using a Resource wrapper pattern.•Modern UI: Fully built with Jetpack Compose for a smooth, reactive user experience.
🏗️ Architecture: Clean ArchitectureThe project is divided into three distinct layers to separate concerns:1. Domain LayerThe core of the app. It is independent of any other layer or library.•Models: Plain Kotlin data classes (e.g., Game).•Repositories: Interfaces defining the data operations.•Use Cases: Individual pieces of business logic (e.g., GetGamesUseCase, GetGameDetailUseCase).
2. Data LayerHandles data retrieval from the network and mapping to domain models.•Remote: Retrofit service and DTOs (Data Transfer Objects).•Mappers: Converts API responses into Domain models.•Repository Implementation: Logic to handle network states and error catching.3. Presentation LayerThe UI layer that reacts to state changes.•MVVM Pattern: ViewModels manage UI state and communicate with Use Cases.•Jetpack Compose: Declarative UI components.•Navigation: Type-safe navigation between screens.
🛠️ Tech Stack•Kotlin: 100% language coverage.•Jetpack Compose: Modern toolkit for building native UI.•Hilt: Dependency Injection for Android.•Retrofit: Type-safe HTTP client for API calls.•Coil: Image loading library for Compose.•ViewModel & StateFlow: To handle UI state in a lifecycle-aware manner.•Coroutines: For asynchronous programming.
🔧 Setup & Installation1. Get an API KeyRegister for a free API key at RAWG.io.2. Add API Key to ProjectAdd your key to the local.properties file (or BuildConfig as configured in your Gradle):properties
RAWG_API_KEY=your_api_key_here
3. Build & Run1.Clone the repository.2.Sync Project with Gradle Files.3.Run on an Android device or Emulator (ensure the emulator supports Google Play Services).🛡️ Error HandlingThe app uses a Resource wrapper to manage UI states:•Success: Data is displayed normally.•Loading: A progress indicator is shown during network calls.•Error: Custom error messages (e.g., "Check your internet connection") are displayed instead of crashing, preventing common SecurityException or InternalServerError issues from breaking the user experience.
